Preventing cascading failures in microservices

1 / 14

Problem Statement

  • In microservice systems, one failing service can bring down everything.
  • Manual monitoring can't react fast enough.
  • No automated way to enforce SLAs in real time.
  • Incident history is easy to tamper with.

What DRCP Does (Solution Overview)

  • Monitors all services in real time.
  • Automatically detects SLA breaches (high latency, high error rate).
  • Trips circuit breakers to stop bad traffic.
  • Records every incident on blockchain (can't be changed).

5 / 14

Microservice Architecture

The system is composed of 4 independent microservices:

1. API Server (cmd/api)

REST endpoints, serves dashboard, manages registry.

2. Telemetry Worker (cmd/worker)

Kafka consumer, budget calculator, policy evaluator.

3. xDS Server (cmd/xds)

Envoy gRPC control plane, pushes circuit breaker configs.

4. Blockchain Anchor (cmd/anchor)

Signs & submits Ethereum transactions for audit.

6 / 14

Data Flow - Telemetry Pipeline

  1. Service sends telemetry POST /api/v1/telemetry
  2. API produces message to Kafka topic telemetry-events
  3. Worker consumes from Kafka
  4. Worker updates Redis sliding window (60s window)
  5. Worker calculates error rate from Redis
  6. Worker sends data to OPA for policy check
  7. If breach → create incident + trip circuit breaker + anchor on blockchain
7 / 14

Key Technical Components

  • Sliding Window (Redis): Uses Redis sorted sets with timestamps. Tracks total requests and errors over last 60 seconds. Uses ZRANGEBYSCORE for queries, ZREMRANGEBYSCORE for cleanup.
  • OPA Policy Engine: Dual mode - built-in rules OR external OPA server. Evaluates error rate and latency against contract thresholds.
  • Envoy xDS: Uses go-control-plane library. Pushes snapshots via gRPC. Trips breaker by setting max connections to 1.
  • Blockchain Anchor: EIP-1559 transactions. Incident hash stored in transaction data field.
8 / 14

Design Patterns Used

  • Event-Driven Architecture: Kafka decouples ingestion from processing.
  • Circuit Breaker Pattern: Envoy auto-trips on SLA breach.
  • Graceful Degradation: Works without Kafka/Redis in reduced mode.
  • Sliding Window: Redis sorted sets for real-time error rates.
  • Policy-as-Code: OPA Rego for flexible SLA rules.
  • Immutable Audit Trail: Ethereum blockchain.
  • Dependency Injection: Handlers receive DB, Kafka, Redis as params.
